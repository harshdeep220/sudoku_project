# Android Sudoku Solver

Automated solver for Sudoku games running on a physical Android device connected via USB. Captures the screen, detects the grid with OpenCV, recognises digits with a lightweight CNN, solves with backtracking, and taps the answer into the app using "quick-fill" mode.

## Prerequisites

- **Python 3.10+**
- **ADB** installed and on your PATH (`adb devices` should list your phone)
- **USB Debugging** enabled on the Android device
- A Sudoku app open on the device with the puzzle visible

## Setup

```bash
cd z:\sudoku_project

# Create virtual environment (optional but recommended)
python -m venv venv
venv\Scripts\activate        # Windows
# source venv/bin/activate   # macOS / Linux

# Install dependencies
pip install -r requirements.txt

# Train the digit-recognition model (one-time, ~30 seconds)
python train_digit_model.py
```

## Usage

```bash
# Auto-detect device
python main.py

# Specify device serial
python main.py --serial DEVICE_SERIAL

# Adjust tap speed (default 0.04s between taps)
python main.py --tap-delay 0.06
```

## Project Structure

| File | Description |
|------|-------------|
| `main.py` | Entry point / orchestrator |
| `device_connector.py` | ADB/UIAutomator2 connection + screen capture |
| `vision.py` | OpenCV grid detection, perspective warp, coordinate mapping |
| `ocr.py` | CNN-based digit recognition |
| `solver.py` | Optimised backtracking Sudoku solver |
| `executor.py` | Quick-fill touch automation |
| `train_digit_model.py` | Synthetic data generation + CNN training |
| `model/digit_model.keras` | Trained model (generated by `train_digit_model.py`) |

## How It Works

1. **Capture** — screenshot the Android device into memory (no temp files)
2. **Detect** — adaptive threshold → find largest contour → perspective warp
3. **Read** — CNN classifies each cell as empty or digit 1–9
4. **Solve** — backtracking with MRV heuristic (typically < 5 ms)
5. **Fill** — quick-fill mode: select number → tap all matching empty cells

## Troubleshooting

- **"Could not detect the Sudoku grid"** — make sure the full 9×9 board is visible on screen with no overlays
- **OCR mis-reads** — re-run `train_digit_model.py` with `SAMPLES_PER_DIGIT = 5000` for more training data
- **Taps land on wrong cells** — adjust `button_delay` and check the button coordinate heuristic in `vision.py`
